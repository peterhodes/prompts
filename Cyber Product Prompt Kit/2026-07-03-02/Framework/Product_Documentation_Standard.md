# Product Documentation Standard

Each product report must contain these sections in order:

1. Document Metadata
2. 10-Minute Refresher
3. Executive Summary
4. Product Positioning
5. Cybersecurity Problems Solved
6. Product Categories
7. Typical Customers and Buying Drivers
8. Core Capabilities
9. Architecture
10. Major Components
11. Data Flow
12. Deployment Models
13. Security Architecture
14. Authentication and Identity
15. Integrations
16. APIs
17. Data Sources
18. Detection Capabilities
19. Automation and Response
20. Incident Response Workflow
21. Logging and Monitoring
22. Scalability
23. High Availability
24. Disaster Recovery
25. Licensing Overview
26. Operational Considerations
27. Best Practices
28. Common Pitfalls
29. Strengths
30. Weaknesses
31. Competitor Comparison
32. Typical Customer Scenarios
33. Meeting Preparation Notes
34. Interview Preparation Notes
35. Questions to Ask Customers
36. Common Misconceptions
37. Future Roadmap, Public Information Only
38. Key Takeaways
39. Research Confidence
40. References

Where information is unavailable, write: `Information not publicly available.` Do not invent details.
