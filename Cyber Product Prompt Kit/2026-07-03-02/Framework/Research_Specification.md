# Research Specification

Use multiple source types:

- Official vendor documentation
- Product documentation
- Architecture guides
- API documentation
- Cloud marketplace listings
- Independent technical blogs
- Analyst reports where available
- Community and practitioner commentary, treated cautiously

Rules:

- Vendor material is useful but not sufficient.
- Prefer current documentation.
- Cite claims where possible.
- Flag uncertainty.
- Separate fact from analysis.
- Do not infer unsupported product capabilities.
