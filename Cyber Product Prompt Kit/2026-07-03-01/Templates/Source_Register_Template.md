## Source reliability guide

### High reliability

Use for technical facts:

- official documentation
- official API documentation
- official integration documentation
- administrator guides
- deployment guides
- release notes
- official security/trust documentation

### Medium reliability

Use for context and comparison:

- recognised analyst reports
- reputable technical blogs
- conference talks
- competitor documentation
- established practitioner commentary

### Lower reliability

Use cautiously:

- community forums
- Reddit
- customer reviews
- reseller summaries
- sales presentations

Lower-reliability sources can be useful for identifying possible pain points, but they should not be treated as definitive without corroboration.
